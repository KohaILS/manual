日誌

記錄追蹤系統的交易內容。您可以使用這些偏好決定要記錄哪些操作以及不記錄哪些操作。日誌可從工具區的 記錄檢視器 查看記錄。

*到此：*其他 > 管理 > 整體系統偏好> 日誌

除錯

ActionLogsTraceDepth

Asks: When logging actions, store a stack trace that goes at most ___ levels deep.

Default: 0

說明：

  • This system preference allows you to store in the action_logs table the information about where in Koha a particular logged action was generated from, and how.

  • Use 0 to deactivate this system preference.

备注

This information is only stored in the database, it is not visible in the log viewer tool.

DumpSearchQueryTemplate

詢問: ___ 將搜尋查詢轉儲為範本參數。

預設值：不要

價：

  • 不要

  • 做

說明：

  • 這個系統偏好允許你查看Zebra或Elasticsearch使用的搜尋查詢，以幫助排除故障。

  • Make sure to enable DumpTemplateVarsIntranet and/or DumpTemplateVarsOpac otherwise this system preference will have no effect.

  • 轉儲的查詢將位於’搜尋_詢問’下的頁面原始程式碼中。

紀錄

AcquisitionLog

詢問：___何時發生採訪行動。

預設：不記錄

值:

  • 不記錄

  • 日誌

說明：

  • 這個系統偏好控制Koha是否會記錄在 :ref:`採訪模組<acquisitions-label>`中進行的各種操作。這些操作包括:

    • 新增新購物籃

    • 重新打開已關閉的購物籃

    • 修改購物籃

    • 修改購物籃標題

    • 修改購物籃使用者

    • 關閉購物籃

    • 批准購物籃

    • 建立新的訂單系列

    • 取消訂單系列

    • 新增發票調整

    • 編輯發票調整

    • 刪除發票調整

    • 根據發票接收一個訂單明细

    • 編輯預算

    • 編輯基金

    • 訂單發布日期（EDIFACT）

  • 這些日誌可以通過 :ref:`日誌查看器工具`<log-viewer-label>進行搜尋。

  • 這些日誌被保存在資料庫中的 action_log 表中。

    • 該模組是’採訪’。

    • 可能採取的行動是

      • 新增_購物籃: a 新購物籃已建立

      • APPROVE_BASKET：EDI 購物籃已獲批准

      • CANCEL_ORDER:：訂單被取消

      • CLOSE_BASKET: 一個 購物籃已關閉

      • 建立_基金: 一個 基金已建立

      • 新增_發票_調整：調整已新增到發票

      • 新增_訂單: 一個 訂單已新增到購物籃

      • 刪除_基金：基金被刪除

      • 刪除_發票_調整：調整已從發票中刪除

      • 編輯_購物籃: 一個 :ref:`購物籃被編輯<edit-basket-label>`（新增或修改訂單）

      • 編輯_購物籃_標題：編輯了購物籃的訊息（例如購物籃名稱或開票地點）

      • 編輯_採購籃_使用者：編輯了採購籃的使用者

      • 編輯_預算：預算已編輯（這不包括 clo）

      • 編輯_基金：基金被編輯

      • 編輯_訂單：訂單已編輯

      • 收到_訂單: 一個 訂單已收到

      • 重新打開_購物籃：關閉的購物籃被重新打開

      • 更新_發票_調整：編輯了對發票的調整

AuthFailureLog

詢問：___ 身份驗證失敗。

預設：不記錄

值:

  • 不記錄

  • 日誌

說明：

  • 此系統偏好控制當讀者未能在 OPAC（或館員介面中的館員）上成功驗證時，Koha 是否應該登入。

  • 這些日誌可以通過 :ref:`日誌查看器工具`<log-viewer-label>進行搜尋。

  • 這些日誌被保存在資料庫中的 action_log 表中。

    • 該模組是’AUTH’

    • 行動是’失敗’

BorrowersLog

Asks: ___ changes to patron records.

預設值：記錄

值:

  • 不記錄

  • 日誌

說明：

  • This system preference controls whether or not Koha should log when a patron’s account is edited.

  • 這些日誌可以通過 :ref:`日誌查看器工具`<log-viewer-label>進行搜尋。

  • 這些日誌被保存在資料庫中的 action_log 表中。

    • The module is ‘MEMBERS’.

    • 可能採取的行動是

      • ADDCIRCMESSAGE: an internal message or OPAC message was added to the patron’s account

      • CHANGE PASS: a patron’s password was changed

      • CREATE: a new patron was added

      • DELCIRCMESSAGE: an internal message or OPAC message was deleted

      • DELETE: a patron’s account was deleted

      • MODIFY: a patron’s account was edited

      • RENEW: a patron’s membership was renewed

备注

啟用此偏好允許追蹤讀者號的變動

CataloguingLog

詢問：___ 書目記錄與館藏記錄的改變。

預設值：記錄

值:

  • 不記錄

  • 日誌

說明：

  • 此系統偏好控制 Koha 是否應在建立、修改或刪除書目記錄或館藏時記錄。

  • 這些日誌可以通過 :ref:`日誌查看器工具`<log-viewer-label>進行搜尋。

  • 這些日誌被保存在資料庫中的 action_log 表中。

    • 該模組是’目錄’。

    • 可能採取的行動是

      • 新增：書目記錄或館藏已建立

      • 刪除：書目記錄或館藏已刪除

      • 修改：書目記錄或館藏已修改，或封面圖片已新增到記錄中

警告

由於每當對標題進行編目或編輯時都會發生這種情況，而且整個記錄都保存在 action_logs 表中，所以它可能非常耗費資源，從而使你的系統變慢或占用大量的空間。
